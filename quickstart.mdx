This guide walks you through creating a new Avalonia application from scratch, installing FluentAvalonia, wiring up the theme, and adding a `NavigationView` with a couple of navigation items. By the end you will have a running app with system-aware light/dark theming and a WinUI-style sidebar.

<Steps>

<Step title="Create a new Avalonia application">

If you do not already have an Avalonia project, create one using the Avalonia templates. Install the templates first if you have not done so:

```bash
dotnet new install Avalonia.Templates
dotnet new avalonia.app -o MyFluentApp
cd MyFluentApp
```

This produces a minimal Avalonia MVVM application. The project file, `MyFluentApp.csproj`, a `App.axaml`, and a `MainWindow.axaml` are all generated for you.

</Step>

<Step title="Install the FluentAvaloniaUI package">

Add the `FluentAvaloniaUI` NuGet package to your project. Run this from inside the project directory:

```bash
dotnet add package FluentAvaloniaUI
```

To pin a specific release, pass the `--version` flag:

```bash
dotnet add package FluentAvaloniaUI --version 2.3.0
```

<Note>
  FluentAvalonia v2.3.0 requires **Avalonia 11.2.5** or later. The Avalonia templates install the latest stable Avalonia, so this requirement is usually satisfied automatically.
</Note>

</Step>

<Step title="Register FluentAvaloniaTheme in App.axaml">

Open `App.axaml` and replace its contents with the following. The key changes are adding the `xmlns:fa` namespace and the `<fa:FluentAvaloniaTheme>` entry in `Application.Styles`.

```xml App.axaml
<Application xmlns="https://github.com/avaloniaui"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             xmlns:fa="using:FluentAvalonia.Styling"
             x:Class="MyFluentApp.App">

    <Application.Styles>
        <fa:FluentAvaloniaTheme PreferSystemTheme="True"
                                PreferUserAccentColor="True" />
    </Application.Styles>

</Application>
```

Setting `PreferSystemTheme="True"` makes the app follow the operating system's light/dark mode automatically. Setting `PreferUserAccentColor="True"` picks up the user's OS accent color as the primary brand color throughout the UI.

<Tip>
  Both properties default to `false`. Omit them if you want to control the theme programmatically instead of following system settings.
</Tip>

</Step>

<Step title="Update App.axaml.cs">

The code-behind file should load the XAML and set up the main window. Replace the contents of `App.axaml.cs` with the following:

```csharp App.axaml.cs
using Avalonia;
using Avalonia.Controls.ApplicationLifetimes;
using Avalonia.Markup.Xaml;

namespace MyFluentApp;

public partial class App : Application
{
    public override void Initialize()
    {
        AvaloniaXamlLoader.Load(this);
    }

    public override void OnFrameworkInitializationCompleted()
    {
        if (ApplicationLifetime is IClassicDesktopStyleApplicationLifetime desktop)
        {
            desktop.MainWindow = new MainWindow();
        }

        base.OnFrameworkInitializationCompleted();
    }
}
```

</Step>

<Step title="Add a NavigationView to MainWindow.axaml">

Open `MainWindow.axaml` and replace its contents with a window that contains a `NavigationView`. The `xmlns:ui` namespace declaration gives you access to all FluentAvalonia controls.

```xml MainWindow.axaml
<Window xmlns="https://github.com/avaloniaui"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:ui="using:FluentAvalonia.UI.Controls"
        x:Class="MyFluentApp.MainWindow"
        Title="My FluentAvalonia App"
        Width="900" Height="600">

    <ui:NavigationView PaneDisplayMode="Left"
                       AlwaysShowHeader="True"
                       Header="My App">

        <ui:NavigationView.MenuItems>
            <ui:NavigationViewItem Content="Home"
                                   Tag="home">
                <ui:NavigationViewItem.Icon>
                    <ui:SymbolIcon Symbol="Home" />
                </ui:NavigationViewItem.Icon>
            </ui:NavigationViewItem>

            <ui:NavigationViewItem Content="Settings"
                                   Tag="settings">
                <ui:NavigationViewItem.Icon>
                    <ui:SymbolIcon Symbol="Settings" />
                </ui:NavigationViewItem.Icon>
            </ui:NavigationViewItem>
        </ui:NavigationView.MenuItems>

        <!-- Page content displayed here -->
        <TextBlock Text="Welcome to FluentAvalonia!"
                   HorizontalAlignment="Center"
                   VerticalAlignment="Center"
                   FontSize="24" />

    </ui:NavigationView>

</Window>
```

The `ui:SymbolIcon` control renders icons from the Fluent symbol font that ships with FluentAvalonia, so no icon font setup is required.

</Step>

<Step title="Run the application">

Build and run the project from the command line:

```bash
dotnet run
```

You should see a window with a left-side navigation pane containing **Home** and **Settings** items, a header bar, and centered welcome text. The app theme follows your OS light/dark preference automatically.

<Tip>
  On Windows 11, `PreferSystemTheme="True"` also picks up high-contrast mode. On Linux, theme detection depends on your desktop environment — KDE, GNOME, and Xfce are supported.
</Tip>

</Step>

</Steps>
